NodesStorage

public final class NodesStorage

NodesStorage is used for fast thread-safe node lookup from any place. Since all nodes is a structures - you can’t just change its properties, so this storage is centralised repository that handles all mutation stuff and notifies delegate when node updates. Also it provides handy things like Reference and Mutator which helps treat nodes as reference types in cases when we need this kind of behaviour. In current ComposerKit implementation this storage is used for pick node to render every frame, but it can be changed in future for performance optimisation advantage. (atomic access of hash table is always slower than direct call.)

  • Reference

    Reference is used for reference-type like treating of nodes managed by NodesStorage that generates reference. Basically it stores node id and have a reference to storage managing this node, and provides API to read and write it. For creating reference - use NodesStorage’s reference(for id: UUID) -> Reference method.
